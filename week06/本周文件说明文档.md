# Week06 文件说明文档

> 本周主题：基于 Week05 结构化聊天机器人，扩展一个可以本地运行的简单 Agent 原型。

---

## 1. 本周目标

Week06 的目标不是使用复杂的外部 Agent 产品，而是在已有聊天机器人基础上理解并实现一个最小可运行的 Agent 流程。

本周 Agent 需要完成的固定流程是：

1. 接收用户输入。
2. 判断是否需要调用工具。
3. 如需要，调用 1 个本地工具获取额外信息。
4. 按固定格式输出最终结果。

固定输出格式为：

```text
结论：
...

依据：
...

是否调用工具：
是 / 否 / 是，但失败
```

## 3. 每个文件的作用

### 3.1 `simple_agent.py`

本周核心实现文件，负责运行简单 Agent。

主要功能：

- 接收用户输入。
- 使用 `should_call_tool()` 判断是否需要读取本地资料。
- 使用唯一工具 `read_local_knowledge()` 读取 `week06/local_knowledge.md`。
- 复用 Week05 的结构化聊天机器人能力组织回答。
- 在模型或依赖不可用时，使用本地 fallback 保证程序仍能返回三段式结果。
- 支持单次命令行运行和交互式运行。

关键函数：

| 函数 | 作用 |
|---|---|
| `should_call_tool(user_input)` | 根据关键词判断是否需要调用工具 |
| `read_local_knowledge(path=None)` | 读取本地知识文件，失败时返回错误结构 |
| `build_agent_prompt(...)` | 将用户问题、工具状态和工具结果组织成提示词 |
| `format_final_answer(...)` | 兜底生成固定三段式输出 |
| `run_agent(...)` | 执行完整 Agent 流程 |
| `stream_agent(...)` | 将完整回答切片，提供流式输出效果 |
| `main()` | 命令行入口 |

运行示例：

```powershell
python week06\simple_agent.py "请根据本地资料说明 Agent 的工作流程"
```

## 5. Agent 完整流程说明

以问题：

```text
请根据本地资料说明 Agent 的工作流程
```

为例，完整流程如下：

```text
用户输入
  -> should_call_tool 判断命中“本地资料 / Agent / 流程”等关键词
  -> 调用 read_local_knowledge 读取 week06/local_knowledge.md
  -> 获取本地资料内容
  -> 组织最终回答
  -> 输出“结论 + 依据 + 是否调用工具”
```

预期输出重点：

```text
结论：
说明简单 Agent 的四步工作流程。

依据：
说明依据来自 week06/local_knowledge.md。

是否调用工具：
是
```

## 6. 失败处理说明

如果本地工具读取失败，例如 `local_knowledge.md` 不存在，Agent 不会崩溃，而是返回：

```text
结论：
未获取到资料。以下是当前可回答部分：...

依据：
本地工具读取失败：<错误信息>

是否调用工具：
是，但失败
```

如果模型或 LangChain 依赖不可用，当前实现会走本地 fallback，仍然保证输出三段式结构。

## 7. 当前实现边界

当前 Agent 可以做：

- 接收一个用户问题或任务说明。
- 判断是否需要本地资料。
- 读取一个固定本地知识文件。
- 根据工具结果组织三段式回答。
- 在工具失败或模型不可用时返回降级结果。

当前 Agent 不做：

- 不做多工具规划。
- 不做联网搜索。
- 不引入向量数据库。
- 不扫描整个项目。
- 不自动修改代码。
- 不执行任意系统命令。
- 不做长期记忆。

## 8. 注意事项

当前实现中保留了 LangChain Agent 分支，并在可用时尝试使用 `create_agent`。不过核心验收流程并不依赖外部 Agent 产品，而是由本地规则判断、本地文件工具和 fallback 共同保证可运行。

如果要完整运行 LangChain Agent 分支，需要确认环境中安装了 `langchain` 主包以及 Week04/Week05 所需依赖；如果依赖不可用，程序会自动退回本地规则流程。
