## 1. 本周目标

在 Week04 LangChain 聊天机器人的基础上，补齐三项能力：

| 目标 | 说明 |
|---|---|
| **稳定系统提示词** | 所有角色/Custom Prompt 末尾强制附加结构化 JSON 输出约定 |
| **输出解析与降级** | 4 级降级策略解析模型输出，解析失败时 fallback 不崩溃 |
| **干净的多轮历史** | 历史消息存储解析后的纯文本，不污染后续轮次上下文 |

**核心原则：** 不修改 Week04 原始代码。所有增强放在 Week05 新模块中，通过导入复用 Week04 的常量与工具函数。

### 2.2 `qa_assistant_structured.py`

**角色:** 主实现模块 — 结构化聊天机器人

**架构:**

```
week04/qa_assistant_lc.py  (🔒 零修改)
       │
       ├── 导入常量: ROLE_PROMPTS, PROVIDER_INFO, DEFAULT_*, ...
       ├── 导入工具: get_model(), _extract_usage(), save_result(), ...
       │
       └── week05/qa_assistant_structured.py  (⚡ 增强模块)
              │
              ├── [新增] OUTPUT_CONTRACT — 结构化 JSON 输出约定文本
              ├── [新增] parse_structured_response() — 4 级降级解析器
              ├── [新增] _extract_json_object() — 手工状态机括号匹配
              ├── [新增] _safe_str(), _safe_confidence() — 类型安全转换
              │
              ├── [重写] build_system_text() — 追加 OUTPUT_CONTRACT
              ├── [重写] build_prompt_template() / build_chat_prompt_template()
              ├── [重写] build_single_shot_chain() / build_chat_chain()
              ├── [重写] handle_session_command() — /role、/system 含约定
              ├── [重写] single_shot_lc() — 集成解析，展示干净 answer
              ├── [重写] interactive_session_lc() — 历史存储纯文本
              │
              └── [重写] run() / main() — CLI 入口
```

**核心函数说明:**

| 函数 | 作用 | 与 Week04 的区别 |
|---|---|---|
| `OUTPUT_CONTRACT` | 追加到每个 system prompt 末尾的格式约定 | 新增 |
| `parse_structured_response(raw_text)` | 解析模型输出 → 7 字段 dict | 新增 |
| `_extract_json_object(text)` | 从任意文本中提取完整 JSON 对象 | 新增（支撑解析器） |
| `build_system_text(role, custom)` | 构建 system prompt | 始终追加 OUTPUT_CONTRACT |
| `single_shot_lc(...)` | 单次问答 | 非流式模式解析后展示 answer 字段 |
| `interactive_session_lc(...)` | 连续对话 | 历史存解析后的纯文本，不存原始 JSON |
| `handle_session_command(...)` | 处理 /role、/system 等指令 | 更新 history[0] 时包含 OUTPUT_CONTRACT |

**CLI 使用方式（与 Week04 完全兼容）：**

```bash
# 单次问答
python qa_assistant_structured.py "什么是机器学习？"
python qa_assistant_structured.py -p cloud --role coder "写一个快排"
python qa_assistant_structured.py --stream "解释 REST API"

# 连续对话
python qa_assistant_structured.py
python qa_assistant_structured.py -p cloud --role teacher

# 会话内指令
/help      /role teacher    /stream      /clear
/provider  /device cpu      /save        /stats
```

## 5. 解析器降级策略

```
模型原始输出 text
  │
  ├─ 策略 1: json.loads(text) 成功且返回 dict     → parse_ok=true
  │
  ├─ 策略 2: 正则匹配 ```json ... ``` 围栏代码块   → parse_ok=true
  │          提取内容后 json.loads
  │
  ├─ 策略 3: 手工状态机括号匹配提取第一个 {...}    → parse_ok=true
  │          提取后 json.loads
  │
  └─ 策略 4: 全部失败                             → parse_ok=false
             answer = 原始文本（CLI 用户仍能看到有用输出）
```

**防御边界：**
- `confidence` 超出 [0, 1] → clamp
- `intent` 非法值（如 "hacking"） → 修正为 "unknown"
- 字段缺失 → 填充默认值
- 字段类型错误（如 answer=123） → `_safe_str()` 转换
- None / 空字符串 → 返回全默认结构
