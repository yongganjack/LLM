# Week07 中文资料问答 Agent 开发计划

> 本文只定义实施任务和验收方式，不包含实现代码。所有改动限定在 `week07/`。

**目标：** 将当前 Week06 复制版改造成一个无需网络和模型服务即可稳定演示的中文“大模型入门课程资料问答助手”。

**方案概述：** 默认使用确定性规则决定“直接回答 / 本地资料检索 / 工具失败降级”。本地资料按 Markdown 二级标题切分，使用中文关键词匹配选出最相关的 1–2 个片段；回答中的依据必须显示资料文件、章节和命中词。每次调用追加一条本地 JSONL 运行记录。

**技术边界：** Python 标准库 + 现有项目依赖；不联网、不引入向量数据库、不要求 DeepSeek/Ollama 可用、不读取或修改 `week03/` 至 `week06/`。

## 2. 实现确定性路由和局部检索

**文件：** 修改 `week07/simple_agent.py`

- [ ] 保留现有的 UTF-8 控制台处理、CLI 入口和三段式兼容能力。
- [ ] 把默认演示路径与模型调用解耦：默认调用不能因为 LangChain、API Key、Ollama 或网络不可用而失败。
- [ ] 定义三类输入路由，并使用可测试的纯函数返回路由结果：
  - 直接回答：问候、`你能做什么`、帮助、使用方式等，不读取资料。
  - 调用工具：询问课程资料中的概念，或显式包含“根据资料”“知识库”“文档”等资料请求词。
  - 范围外：不属于课程资料且不是帮助类问题，直接说明能力边界，不伪造资料依据。
- [ ] 将当前“读取整份文档”的工具改为“读取资料后按 Markdown 二级标题切分并检索”。
- [ ] 检索规则采用稳定、可解释的中文关键词匹配：提取查询中的有效词，按标题命中优先、正文命中次数其次排序，返回 Top 1–2 个片段。
- [ ] 当无章节命中但资料读取成功时，返回“资料未覆盖”的正常结果；工具状态仍需如实反映是否读取过资料。
- [ ] 为每个检索结果保留以下可展示元数据：相对来源路径、章节标题、命中词、片段序号。
- [ ] 避免在默认演示路径中让 LLM 自主决定是否调用工具；保留原有 LangChain/Week05 能力仅作为可选增强或兼容代码，不能改变确定性状态。

完成判据：同一输入连续运行三次，路由、引用章节和工具状态完全一致。

## 3. 统一回答、依据和失败输出

**文件：** 修改 `week07/simple_agent.py`

- [ ] 将最终输出固定为四段，前三段保持现有名称以兼容旧逻辑：
  - `结论：`
  - `依据：`
  - `是否调用工具：`
  - `运行记录：`
- [ ] 工具成功时，“依据”必须至少包含：`week07/local_knowledge.md#章节名`、命中词和“本地关键词检索”说明。
- [ ] 直接回答时，“依据”必须明确“未调用本地资料工具”，并说明其是帮助/能力说明或范围边界回答。
- [ ] 资料读取失败时，结论必须以“未获取到资料”说明失败；依据必须展示可读的错误原因；工具状态必须为“是，但失败”。
- [ ] 资料读取成功但无相关章节时，不得虚构章节或来源；结论应说明“当前资料未覆盖该问题”。
- [ ] 保持空输入可用：提示用户输入有效问题，不调用工具，不写入错误的来源信息。
- [ ] 在最终输出前使用统一格式化函数补齐遗漏段落，禁止让异常栈直接输出给用户。

完成判据：任何输入的返回文本均含四个段落标题；失败路径不崩溃、不隐藏错误、不声称已获得资料。

## 4. 增加可安全演示的失败开关和运行记录

**文件：** 修改 `week07/simple_agent.py`；新增 `week07/.gitignore`、`week07/runs/.gitkeep`

- [ ] 增加 CLI 开关 `--simulate-tool-failure`，只对本次进程模拟本地工具失败。
- [ ] 该开关不能重命名、删除或修改真实知识文件；它应返回确定的模拟错误文本，方便现场演示。
- [ ] 为每次 Agent 调用生成 `run_id`，使用本地时间戳并保证同进程内不重复。
- [ ] 在 `week07/runs/` 追加 JSONL 记录；每条至少包含：时间、run_id、用户输入、路由、工具是否调用、工具状态、来源/章节、命中词、错误信息、回答摘要和日志写入状态。
- [ ] 日志文件按日期或固定文件名组织，但必须是一行一个 JSON 对象，UTF-8 编码。
- [ ] 将 `runs/*.jsonl` 写入 `week07/.gitignore`；用 `runs/.gitkeep` 保留空目录。
- [ ] 如果日志目录创建或写入失败，主回答仍须成功返回；“运行记录”段要如实显示未写入原因。

完成判据：正常、范围外和模拟失败三次调用各新增一条合法 JSONL；禁用日志目录写入的测试中仍能得到完整四段回答。
