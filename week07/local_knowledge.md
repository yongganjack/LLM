## 提示词模板

提示词模板（Prompt Template）是大模型应用中最基础的组件之一。它是把固定的系统要求或指令框架与动态的用户输入组合起来，形成一个完整的、可直接发送给模型的提示词（Prompt）。

一个典型的提示词模板包含三个部分：

1. **系统指令：** 固定文本，定义模型的角色、行为规则和输出格式。例如"你是一个专业的中文问答助手"。
2. **上下文占位符：** 用于插入外部知识、对话历史或检索结果的位置，通常用 `{context}` 或 `{history}` 标记。
3. **用户输入占位符：** 用于插入用户当前问题的位置，通常用 `{question}` 或 `{input}` 标记。

提示词模板的作用：
- 统一管理提示词结构，避免每次调用时手工拼接。
- 把动态内容与静态规则分离，便于维护和调试。
- 为不同的下游任务提供一致的输入格式。

常见框架（如 LangChain）提供了 `ChatPromptTemplate` 和 `MessagesPlaceholder` 等组件来构建提示词模板，支持多轮对话历史的自动管理。

## 上下文窗口

上下文窗口（Context Window）是指大语言模型一次可以处理的最大 token 数量，包括输入（prompt）和输出（completion）的总和。

关键概念：
- **Token：** 文本被模型分词器切分后的最小处理单元。中文约 1—2 个字符为一个 token，英文约 0.75 个单词为一个 token。
- **输入 Token：** 发送给模型的提示词所消耗的 token 数量，包括系统指令、对话历史、检索到的上下文等。
- **输出 Token：** 模型生成回答所消耗的 token 数量，受 `max_tokens` 参数限制。
- **上下文窗口限制：** 如果输入加上输出超过了模型的上下文窗口，最前面的内容会被截断，可能导致模型"遗忘"早期信息。

实际影响：
- 长文档或长对话需要在接近窗口上限时做摘要或截断。
- 多轮对话中，超过上下文窗口的历史消息会被丢弃，影响连贯性。
- 检索增强生成（RAG）可以减少对超长上下文的依赖，只把最相关的片段放入上下文窗口。

## RAG 与本地检索

RAG（Retrieval-Augmented Generation，检索增强生成）是一种在生成回答之前先从知识库中检索相关信息的架构模式。

基本流程：
1. **索引阶段：** 把文档切分成片段，通过 Embedding 模型转为向量，存入向量数据库。
2. **检索阶段：** 收到用户问题后，把问题也转为向量，在向量数据库中找到最相似的 k 个文档片段。
3. **生成阶段：** 把检索到的片段作为上下文，与用户问题一起送入大模型，生成基于资料的准确回答。

本地检索是本项目 Week07 使用的一种简化版 RAG 思路：
- 不使用向量数据库或 Embedding 模型。
- 不依赖外部搜索 API。
- 使用可解释的中文关键词匹配：按 Markdown 二级标题切分资料，标题命中优先排序，正文命中次数其次。
- 每次返回最相关的 1—2 个资料片段，并保留来源章节和命中词信息。
- 完全不依赖网络连接或模型服务即可完成检索。

RAG 的优势：
- 让模型基于外部知识库回答，减少幻觉。
- 知识可以独立更新，无需重新训练模型。
- 可以溯源每个回答的依据片段。

## Agent 与工具调用

在大模型应用中，Agent 是指能够自主决策、调用工具、规划步骤来完成复杂任务的智能体。

Agent 的核心循环（ReAct 模式）：
1. **思考（Reason）：** 分析用户输入和当前状态，决定下一步行动。
2. **行动（Act）：** 调用合适的工具（如搜索、计算、文件读取），获取外部信息。
3. **观察（Observe）：** 接收工具返回的结果，更新自身状态。
4. **重复或结束：** 如果信息足够，生成最终回答；否则继续思考和行动。

工具（Tool）是 Agent 与外部世界交互的接口：
- 每个工具有明确的输入、输出和功能描述。
- Agent 根据工具描述和当前需求自主决定是否调用、何时调用。
- 常见的工具包括：搜索引擎、计算器、代码解释器、文件读写、API 调用等。

本项目的 Agent 设计原则：
- 只有一个工具：读取本地知识文件。
- 使用确定性规则（非 LLM 决策）判断是否需要调用工具。
- 工具调用失败时不崩溃，返回明确的失败状态和降级结果。
- 输出格式固定为四段：结论、依据、是否调用工具、运行记录。

## 使用边界

本 Agent（Week07 中文资料问答助手）的能力边界如下：

**能做什么：**
- 回答大模型入门课程资料中已覆盖的概念问题。
- 根据用户问题检索本地资料，展示依据来源（文件、章节、命中词）。
- 在资料未覆盖时诚实说明范围限制。
- 在工具读取失败时给出降级结果，不崩溃、不虚构。
- 每次调用记录运行日志，方便追踪和验收。

**不能做什么：**
- 不能联网搜索或访问外部链接。
- 不能读取 Week07 以外的任何文件。
- 不能修改、删除或重命名任何文件。
- 不能执行系统命令或脚本。
- 不能调用多个工具进行复杂规划。
- 不能实现长期记忆或多会话上下文保持。
- 不能代替真实的大模型进行内容生成或推理（本地确定性逻辑只负责检索和格式化）。

**资料未覆盖时：**
Agent 应如实说明"当前资料未覆盖该问题"，不虚构来源、不伪造章节、不声称资料包含不存在的内容。用户可以查阅"资料来源说明"中列出的外部链接进行延伸学习。
