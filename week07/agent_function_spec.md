# Week07 中文资料问答 Agent — 功能说明

## 概述

Week07 中文资料问答 Agent 是一个模型驱动的课程资料问答助手。课程问题会由本地确定性规则完成意图识别、资料检索、重排和上下文组装，再调用所选 provider 的模型服务生成答案；引用、工具状态与日志始终由本地逻辑生成。

课程问答的主工作流如下：

```text
用户提问
  ↓
问题理解与意图识别
  ↓
检索课程资料
  ↓
筛选/重排相关内容
  ↓
组织上下文
  ↓
调用大模型生成答案
  ↓
补充引用/来源
  ↓
返回给用户
```

问候、帮助、空输入、范围外问题和资料读取失败属于例外路径：它们会在对应阶段提前返回，不进入课程资料检索与模型生成。

---

## 能做什么

- 回答大模型入门课程资料中已覆盖的概念问题，包括：
  - 提示词模板（Prompt Template）
  - 上下文窗口（Context Window）
  - RAG 与本地检索
  - Agent 与工具调用
  - 使用边界
- 根据用户问题在本地课程资料中检索相关片段，并以可验证的方式展示依据（文件、章节、命中词）。
- 回答问候、帮助请求和使用方式说明。
- 在资料未覆盖时诚实说明范围限制。
- 在工具读取失败时给出降级结果，明确显示"未获取到资料"和"是，但失败"。
- 在模型服务不可用时保留检索依据，并明确说明未生成最终答案。
- 每次调用记录一条 JSONL 运行日志，方便追踪和验收。
- 在 CLI 中按终端宽度自动折行长段落，保留四段标题、已有空行和原始 `run_agent()` 返回值。

---

## 不能做什么

- 不能联网搜索、抓取网页或访问用户提供的外部链接；课程问答仅会调用所选 provider 的模型 API。
- 不能读取 `week07/` 以外的任何文件。
- 不能修改、删除或重命名任何文件。
- 不能执行系统命令或脚本。
- 不能调用多个工具进行复杂任务规划。
- 不能实现长期记忆或多会话上下文保持。
- 不能对课程资料之外的事实进行核验或推理。

---

## 何时直接回答

以下情况 Agent 不读取本地资料，也不调用模型，直接返回固定说明：

- 用户问候（"你好"、"hello"等）。
- 用户询问 Agent 能力或使用方式（"你能帮助我做什么？"、"怎么使用？"等）。
- 用户输入为空或仅含空白字符。
- 用户问题被路由为"范围外"（不属于课程资料主题且不是帮助类问题）。

---

## 何时调用工具

以下情况 Agent 读取本地资料并检索回答：

- 问题涉及课程资料主题关键词：提示词模板、上下文窗口、RAG、检索增强、Agent、工具调用、使用边界等。
- 问题显式包含资料请求词："根据资料"、"知识库"、"文档"、"课程资料"、"本地资料"等。
- 问题涉及大模型入门基础概念：prompt、token、embedding、向量、微调、预训练等（需命中至少 2 个弱匹配词）。

读取成功且命中 1—2 个相关片段后，Agent 会将重排后的片段作为上下文调用模型。模型只生成“结论”正文；资料来源、章节、命中词、工具状态与运行记录由本地代码补充。资料可读但无命中时，Agent 明确说明资料未覆盖该问题，不调用模型。

---

## 本地知识来源

- 唯一运行时知识来源：`week07/local_knowledge.md`
- 资料以 Markdown 二级标题（`## `）组织，共 5 个主题章节加 1 个前言/来源说明。
- 检索阶段只读取本地资料文件，不访问外部文件或搜索引擎；课程答案生成阶段会调用所选模型服务，默认 provider 为 `cloud`。

资料整理的参考链接（仅用于编写摘要，运行时不可访问）：
- https://zhuanlan.zhihu.com/p/2000239845751137469
- https://github.com/datawhalechina/llm-universe
- https://github.com/datawhalechina/llm-cookbook

## 检索方式

使用**可解释的中文关键词匹配**：

1. 清洗查询文本：去除标点、空白和常见停用词（"的"、"什么"、"如何"等）。
2. 提取查询中的字符 bigram（双字组合）作为匹配单元。
3. 对每个资料片段（按 `## ` 标题切分）计算匹配分数：
   - **标题命中**：每个匹配 bigram 权重 ×3
   - **正文命中**：每个匹配 bigram 按出现次数权重 ×1
4. 按分数降序排列，返回最相关的 1—2 个片段。

此检索和重排方法不依赖向量数据库、Embedding 模型或搜索服务，结果完全可解释和可复现；后续答案生成由模型服务负责。

---

## 回答中的依据格式

工具成功时，依据段格式如下（以提示词模板为例）：

```text
本地来源：week07/local_knowledge.md#提示词模板；
检索方式：本地关键词检索（标题命中优先 ×3，正文命中次数 ×1）；
命中词：提示词、模板。
```

多个命中章节时，来源引用逐一列出。无命中时，依据段说明"未找到与问题相关的章节"。

---

## 回答结构（四段式）

所有 `run_agent()` 返回结果均包含以下四段：

```text
结论：
<中文回答正文>

依据：
<资料来源、章节、检索方式、命中词；或未调用工具/失败原因>

是否调用工具：
<是 / 否 / 是，但失败>

运行记录：
<run_id 与日志写入位置，或日志失败原因>
```

工具状态仅描述本地资料工具：资料检索成功但模型调用失败时，状态仍为`是`，结论会说明“模型服务不可用，未生成最终答案”。

## 日志文件

- 位置：`week07/runs/agent_runs.jsonl`
- 格式：每行一个 JSON 对象，UTF-8 编码
- 每条记录包含以下字段：

| 字段 | 说明 |
|---|---|
| 时间 | 北京时间，格式 `YYYY-MM-DD HH:MM:SS` |
| run_id | 唯一运行标识符，格式 `run_YYYYMMDD_HHMMSS_<uuid8>_<counter>` |
| 用户输入 | 用户原始输入（最多 500 字） |
| 路由 | `direct_answer` / `local_retrieval` / `out_of_scope` |
| 是否调用工具 | `true` / `false` |
| 工具状态 | `是` / `否` / `是，但失败` |
| 来源 | 资料文件相对路径 |
| 章节 | 命中的章节标题 |
| 命中词 | 匹配到的关键词 |
| 错误信息 | 工具读取、资料解析或模型调用失败的错误描述（已脱敏） |
| 回答摘要 | 回答内容的前 200 字 |
| 日志写入状态 | `成功`（日志记录实际写入时） |

- 日志写入失败不影响主回答；"运行记录"段会如实说明失败原因。
- 日志文件已在 `.gitignore` 中忽略（`runs/*.jsonl`）。
- 每次 Agent 调用最多写入一条 JSONL 记录；来源字段只使用相对路径 `week07/local_knowledge.md`。
- 日志写入失败时不会产生半条或占位记录；回答仍返回四段格式，并在“运行记录”段说明失败原因。

## CLI 参数

```text
用法: python week07/simple_agent.py [选项] [问题文本]

选项:
  --no-stream              禁用流式输出（单次模式推荐）
  --simulate-tool-failure   模拟工具读取失败（仅本次调用）
  -p, --provider <name>    模型后端 (cloud/local)，默认 cloud
  --stream                 启用流式输出

无参数启动进入交互模式。
```

`--simulate-tool-failure` 仅在当前进程内模拟工具读取失败，绝不重命名、删除或改写真实资料文件。退出后真实文件仍可正常读取。

单次模式和交互模式的终端输出都会自动按当前终端宽度折行；该展示处理不改变 API 返回文本、引用内容或 JSONL 日志。

---

## 兼容性说明

- 保留原有 `run_agent()`、`stream_agent()`、`read_local_knowledge()`、`format_final_answer()` 等函数签名。
- 新增参数均带有默认值，旧调用方式不受影响。
- 课程问答默认复用 Week05 的模型链与 Week04 的模型工厂，依赖所选 provider 的服务和凭证。
- 问候、帮助、范围外说明和 `--simulate-tool-failure` 不调用模型；课程问答的模型故障会返回四段式失败结果，而不会用本地摘录冒充模型答案。
